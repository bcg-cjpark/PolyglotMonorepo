# v2.0.0 — `example-*` 접두 제거 & 템플릿 generic 화 (2026-04-27)

> v1.0.0 출고 후 첫 마이너 사이클을 돌리지 않고 곧장 메이저로 점프한 이유는, 템플릿 사용자에게
> "`Use this template` → `/bootstrap` 후에도 `apps/example-*` 가 잔존" 한다는 가장 큰 마찰을
> 디렉토리/이름 레벨에서 한 번에 끊어내기 위해서다. 이 릴리즈 이후 `apps/api` / `apps/web` 이
> 템플릿의 정식 디렉토리 이름이 된다.

## 하이라이트

- **`apps/example-api` → `apps/api`**, **`apps/example-web` → `apps/web`** 일괄 rename. Nx project name, Spring `application.name`, Gradle `rootProject.name` 까지 모두 generic 역할 이름으로 통일.
- **부트스트랩 단순화** — `/bootstrap` 스킬에서 옵션적 디렉토리 rename(§ 3.3) 단계가 통째로 사라짐. 42개 파일을 매번 정확히 sed 해야 하는 brittle 한 단계가 없어졌고, 에이전트 프롬프트도 고정 경로(`apps/api`, `apps/web`) 가정 가능.
- **템플릿 리셋 단계 신설** — `/bootstrap` 이 본인 레포의 릴리즈 흔적까지 리셋 (`CHANGELOG.md` → 빈 `[Unreleased]` 셸, `docs/releases/v*.md` 삭제). 사용자가 `Use this template` 후 깨끗한 상태로 출발.
- **README 템플릿 안내 섹션** — 최상단에 "템플릿으로 사용하기" (Use this template → `/bootstrap` 흐름) 추가, 신규 사용자가 1차 진입점에서 즉시 다음 행동을 알 수 있게.
- **추가 앱 생성 규약은 유지** — `nx g @monorepo/generators:api-app|web-app` 으로 specific 이름(`billing-api`, `admin-web` 등) 부여하는 패턴은 그대로. 변경된 건 *템플릿 기본 두 앱의 이름*.

## Breaking changes & 마이그레이션

### 무엇이 바뀌었나

| Before (v1.x) | After (v2.0.0) |
|---|---|
| `apps/example-api/` | `apps/api/` |
| `apps/example-web/` | `apps/web/` |
| `pnpm nx run example-api:build` | `pnpm nx run api:build` |
| `pnpm nx run example-web:build` | `pnpm nx run web:build` |
| Spring `spring.application.name=example-api` | `spring.application.name=api` |
| Gradle `rootProject.name = "example-api"` | `rootProject.name = "api"` |

추가로 변경 영향이 미친 곳: `.claude/agents/**` 의 모든 경로 참조, `CLAUDE.md` 작업 검증 규칙 표, `docs/tech-stack/**`, `tools/generators/{api-app,web-app}/generator.ts` 의 기본 출력 경로, `.gitignore`, 그리고 모든 PRD/screen 문서 내 명시된 경로 표기.

### 마이그레이션 절차

이전 v1.x 베이스 위에서 작업 중이었다면 다음 4단계로 v2.0.0 위로 끌어올린다:

```bash
# 1. 텍스트 참조 일괄 치환 (lock 파일과 v1 릴리즈 노트는 보존)
git ls-files | grep -v -E '^(pnpm-lock\.yaml|docs/releases/)' | \
  xargs sed -i 's/example-api/api/g; s/example-web/web/g'

# 2. 디렉토리 rename (Git 이력 보존을 위해 git mv)
git mv apps/example-api apps/api
git mv apps/example-web apps/web

# 3. pnpm-lock.yaml 재생성 (project name 이 바뀌었으므로 락 키도 갱신 필요)
pnpm install

# 4. 검증 — 모두 PASS 해야 함
pnpm nx run api:build
pnpm nx run api:lint
pnpm nx run web:build
pnpm nx run web:lint
```

### 의도적으로 보존된 것

- `docs/releases/v1.0.0.md` — v1.0.0 시점의 디렉토리 표기 (`apps/example-api`, `apps/example-web`) 가 그대로 남아 있다. **역사적 기록이므로 손대지 않는다.**
- `pnpm-lock.yaml` — sed 대상에서 제외. `pnpm install` 이 새 project name 기준으로 자동 재생성한다.

### `/bootstrap` 스킬 변경 사항 (요약)

| 단계 | v1.x | v2.0.0 |
|---|---|---|
| § 3.3 디렉토리 rename | 옵션 단계 (`apps/example-* → apps/<name>-*`) | **제거** (디렉토리는 이미 generic) |
| stale 예제 PRD 제거 | `rm docs/prd/user-*.md` 등 | **제거** (ba04012 에서 이미 비워짐) |
| 템플릿 릴리즈 흔적 리셋 | 없음 | **신설** — `CHANGELOG.md` 를 빈 `[Unreleased]` 셸로 / `docs/releases/v*.md` 삭제 |
| dup-check grep 키워드 | `example-api` / `example-web` 등 다중 | `com.example.template` 단일 토큰 |
