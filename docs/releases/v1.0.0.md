# v1.0.0 — 첫 공식 릴리즈 (2026-04-24)

## 하이라이트

- **에이전트 파이프라인 완성**: 7 개 팀 + 기획팀 4 팀원 + 각 팀장. 메인 에이전트가 팀원들을 호출해 PRD → (시안) → 백엔드 ∥ UI → 프론트 → e2e → 통합 → 감사를 순차 돌리는 구조.
- **`@monorepo/ui` primitive 전집 + Storybook + 런타임 테스트**: Modal async onConfirm/onCancel, Table primitive, Light/Dark 토큰 자동 스위칭.
- **파이프라인 [2b] 화면 시안 프로세스**: Stitch 없이도 `screen-concepter` 가 `libs/ui` 제약 반영한 시안 HTML 2~4 개 + 3 레이어 비교 인덱스(index / compare-\*/ variant 본체) 생성.
- **프로젝트 모드 (inhouse / external / mock)**: `docs/tech-stack/backend.md` 헤더가 단일 진실 소스. 모드별 활성 팀 분기 + MSW lazy 설치 정책.
- **dogfooding 완주로 실증**: 제로 베이스에서 User/Todo/Memo 3 피처를 파이프라인 [1]~[9] 로 풀 사이클 돌리고, 그 과정에서 발견된 6 개 gap (Flyway 재기동 / Security 병렬 / Jackson PUT / Modal 양보 / 에이전트 세션 로드 / Tailwind alias) 전부 규약/코드에 반영.

## ❗ Breaking Changes & 마이그레이션

첫 공식 릴리즈라 이전 버전 대비 "breaking" 개념이 없으나, 템플릿을 **fork/use template 후 기존에 사용하던 사용자** 가 있다면:

- **패키지 이름 규약**: `@monorepo/*` 접두사가 고정. `/bootstrap` 스킬 또는 수동 치환으로 프로젝트별 네임스페이스로 변경.
- **`mode: inhouse`** 가 기본값. 프론트만 개발할 거면 `mode: external` 또는 `mock` 로 수동 변경 후 `/bootstrap` 의 관련 셋업.
- **DB 엔진**: MySQL 8.4 LTS 전제. PostgreSQL 로 되돌리려면 `docs/tech-stack/backend.md` 개정 + Flyway V1 재작성 + 타입 매핑 교체.
- **Flyway V1 미존재**: v1.0.0 은 셸 상태라 migration 이 없음. 첫 피처 백엔드를 구현할 때 `V1__<name>.sql` 부터 작성.
