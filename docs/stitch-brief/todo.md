## 프로덕트 컨텍스트

개인 할일 관리 도구. 사용자는 할 일을 쌓아두고 한 번의 상호작용으로 완료/미완료를 토글하며,
상태(전체 / 진행 중 / 완료) 로 좁혀 보고 마감일을 추적한다. 톤은 깔끔하고 리듬감 있는 목록 —
개인 생산성 도구 답게 시각 노이즈 최소화, 읽기 쉬운 행 간격, 완료된 항목은 시각적으로 뒤로
물러나게. V1 은 단일 사용자 전제의 전역 공유 리스트이며 카테고리/우선순위/알림 같은 부가 축은
없다.

## 전체 페이지 리스트

### 1. TodoListPage

- **페이지 이름**: `TodoListPage`
- **Route**: `/todos`
- **목적**: 할일 목록을 상태별로 좁혀 보고, 한 번의 상호작용으로 완료 토글·삭제·생성 진입을 할 수 있다.
- **필요 UI 카테고리** (구현체 언급 금지):
  - 페이지 헤더 — 페이지 제목 ("할 일") + 우측 Primary 버튼 ("+ 새 할 일")
  - 상태 필터 세그먼트 컨트롤 (또는 라디오 그룹 언더라인 스타일) — "전체 / 진행 중 / 완료" 3탭
  - 정형 스키마 표 (중·소량 행) — 컬럼: 체크 토글 / 제목 / 마감일 / 상태 배지 / 액션
  - 체크박스 (행 내 토글용)
  - 상태 배지 또는 칩 — "진행 중" / "완료" / "기한 지남"
  - 아이콘 버튼 또는 텍스트 버튼 — 행 내 "편집" / "삭제"
  - 빈 상태 안내 문구 ("표시할 할 일이 없습니다" — 필터별 문구는 구현 단계 재량)
  - 로딩 플레이스홀더 (행 자리 스켈레톤 또는 "불러오는 중…" 텍스트)
  - 에러 안내 + 재시도 버튼
- **주요 인터랙션**:
  - 상태 필터 탭 클릭 → 서버에 `status=all|active|completed` 쿼리를 다시 보냄 (클라이언트 필터 금지).
  - 행의 체크박스 클릭 → 완료/미완료 즉시 토글 (낙관적 업데이트).
  - 행의 "편집" 클릭 → `/todos/:id/edit` 이동. "삭제" 클릭 → 확인 후 hard delete.
  - 헤더 "+ 새 할 일" 클릭 → `/todos/new` 이동.
  - 완료된 항목: 제목에 취소선 + 의미 토큰(muted 계열) 으로 흐리게.
  - overdue 항목 (마감일 과거 + 미완료): 마감일 텍스트를 위험 색상 토큰(`font-color-danger` 계열) 으로 구분.
- **데이터 바인딩 대상 API**:
  - `GET /todos?status=all|active|completed` — 리스트 조회 (정렬 `createdAt DESC`).
  - `PATCH /todos/{id}/toggle` — 완료 토글 (낙관적 업데이트 대상).
  - `DELETE /todos/{id}` — hard delete.

### 2. TodoFormPage

- **페이지 이름**: `TodoFormPage`
- **Route**:
  - `/todos/new` — 생성 모드
  - `/todos/:id/edit` — 편집 모드
- **목적**: 단일 할 일의 제목과 마감일을 생성하거나 편집한다.
- **필요 UI 카테고리** (구현체 언급 금지):
  - 페이지 헤더 — 페이지 제목 ("새 할 일" / "할 일 편집") + (선택) 뒤로가기 진입점
  - 폼 영역
    - 단일행 텍스트 입력 — 제목 (필수, 최대 200자, 공백만 금지, **한글 IME 대상**)
    - 날짜 선택 입력 — 마감일 (선택, 비우면 기한 없음)
    - (편집 모드에서만) 완료 여부 체크박스 — `PUT` payload 의 `completed` 필드용
  - 필드 에러 문구 — 의미 토큰(`font-color-danger` 계열)
  - 액션 버튼 행 — Primary 버튼 ("저장") + 보조/텍스트 버튼 ("취소")
  - 로딩 플레이스홀더 (편집 모드 초기 fetch)
  - 에러 안내 + 재시도 버튼
- **주요 인터랙션**:
  - "저장" 클릭 → 생성 모드는 `POST /todos`, 편집 모드는 `PUT /todos/{id}`. 성공 시 `/todos` 로 복귀.
  - "취소" 클릭 → `/todos` 로 복귀.
  - 제목 공백 / 200자 초과 시 필드 에러 표시, 저장 버튼은 유효해질 때까지 동작 불가 (구체 disabled 조건은 구현 단계).
  - 마감일 입력 비우고 저장 → payload `dueDate: null` 로 전송 (편집 모드에서 기한 제거도 동일).
- **데이터 바인딩 대상 API**:
  - `GET /todos/{id}` — 편집 모드 초기 fetch.
  - `POST /todos` — 생성.
  - `PUT /todos/{id}` — 전체 교체 (모든 필드 필수, `dueDate: null` 명시로 기한 제거).

## 디자인 제약

- **토큰 기반** — 색/간격/타이포 모든 시각 값은 `libs/tokens` + Tailwind 토큰 유틸 경유. 하드코딩 `#hex` / `px` 색상 리터럴 금지.
- **Light/Dark 양쪽 대응** — 단일 테마만 렌더 확인 후 확정 금지.
- **글로벌 팔레트 준수** — Primary indigo (`#4f46e5` 계열, `800`=앵커) / Secondary amber. 한 화면에 Primary fill CTA 는 1개 원칙.
- **헤더 유지 원칙** (`global-states.md`) — Loading / Error / Empty 세 상태 모두 페이지 헤더 (제목 + "+ 새 할 일") 는 항상 렌더. 본문만 상태 뷰로 치환. 페이지 전체를 `"Loading…"` 한 줄로 대체하는 패턴 금지.
- **표 primitive 규칙** (`data-display.md`) — 정형 스키마 + 체크박스 컬럼 + 액션 컬럼 패턴. 행 내 액션은 표의 셀 렌더 콜백 안에서 기존 UI 카테고리 (체크박스, 버튼, 배지) 로 합성. native `<table>` / `<button>` / `<input>` 금지.
- **완료된 항목 시각 구분** — 의미 토큰(`text-muted` 계열) 으로 흐리게 + 제목 취소선. 스케일 직접 참조(`neutral500` 등) 금지 — semantic 토큰만.
- **overdue 시각 구분** — 마감일이 과거이고 `completed=false` 인 항목: 마감일 텍스트에 위험 색상 토큰(`font-color-danger` 계열). 행 전체를 빨갛게 칠하지 말 것 (읽기 방해).
- **상태 필터는 세그먼트 컨트롤 또는 라디오 그룹(언더라인)** — 드롭다운 금지 (옵션 3개라 오버). 현재 선택된 탭은 Primary anchor 로 강조.
- **한글 IME 대상 필드** — `TodoFormPage` 의 제목 입력 (`composition*` 이벤트 처리가 필요한 필드).
- **표 행 hover** — 헤더 배경과 구분되는 미묘한 톤 (`data-display.md` §2.1 참조). 강조 세기 과도하지 말 것.

## 비포함 (Out of scope)

Stitch 가 아래 요소를 자발적으로 만들어내면 **무시**하고 화면에 반영하지 않는다.

- 카테고리 / 태그 분류 UI
- 우선순위 (High / Medium / Low) 표시
- 서브태스크 / 체크리스트 중첩
- 드래그앤드롭 정렬 핸들
- 알림 / 리마인더 설정 UI
- 페이지네이션 UI (V1 미적용 — 전체를 단일 응답으로 반환)
- 검색 입력
- 컬럼 정렬 / 컬럼 필터 (표 primitive 가 비기능으로 제외)
- 대량 선택 / 일괄 작업 툴바
- soft delete / 복구 / 휴지통 UI
- 다중 사용자 / 공유 / 멘션 (V1 단일 사용자 전제)
- 첨부 파일 / 이미지 / 리치 텍스트
