## 프로덕트 컨텍스트

제목과 본문으로 구성된 간단한 텍스트 메모를 한 공간에 모아두고 최신순으로 훑어볼 수 있게 하는 개인 노트 기능이다. 사용자는 목록 화면에서 메모를 빠르게 작성하고, 기존 메모를 열람/수정/삭제한다. 톤은 "편안하고 가독성 높은 개인 노트" — 카드 단위로 숨을 쉬는 여백, 본문 스니펫 위주의 정적인 인상. 화려한 테이블/그리드 레이아웃이 아니다. 본문은 plain text 로만 렌더하며 마크다운/HTML 은 보여주지 않는다.

## 전체 페이지 리스트

---

### 1. MemoListPage

- **페이지 이름**: `MemoListPage`
- **Route**: `/memos`
- **목적**: 메모 목록을 최신순 카드 리스트로 보여주고, 신규 작성 / 상세 열람 / 편집 / 삭제 흐름의 진입점 역할을 한다.
- **필요 UI 카테고리** (구현체 언급 금지):
  - **페이지 헤더**: 페이지 제목 ("메모") + 우측 정렬된 Primary 버튼 ("+ 새 메모"). Loading/Error/Empty 세 상태 모두에서 유지.
  - **목록 + 합성 카드** (data-display.md §3 매트릭스의 "목록 + 합성 카드" 범주 적용). 각 카드는 다음 구성:
    - 제목 (단일 라인, 길면 말줄임)
    - 본문 스니펫 (2~3줄, plain text, 말줄임)
    - 작성일 / 수정일 메타 텍스트 (보조 톤)
  - **페이지네이션 내비게이션**: 목록 하단. "이전" / "다음" 방향 버튼 + 현재 페이지/총 페이지 인디케이터 (예: `3 / 12`). 기본 size=20.
  - **Loading 본문 뷰**: 카드 모양 스켈레톤 플레이스홀더를 페이지 사이즈만큼 반복. 헤더는 그대로.
  - **Error 본문 뷰**: 중앙 정렬 에러 안내 텍스트 (위험 톤) + "다시 시도" Secondary 버튼. 헤더는 그대로 (+ 새 메모 버튼 살아있음).
  - **Empty 본문 뷰**: 중앙 정렬 안내 텍스트 ("아직 메모가 없어요" 류). 본문 내 CTA 는 선택 — 헤더의 "+ 새 메모" 버튼이 1차 경로이므로 본문은 안내 텍스트만으로 충분. (global-states.md §3.3)
- **주요 인터랙션**:
  - "+ 새 메모" 클릭 → **신규 작성 모달** (오버레이 컨테이너) 오픈.
  - 카드 클릭 → **상세 모달** 오픈 (읽기 전용).
  - 하단 "이전" / "다음" → 페이지 이동. 첫/마지막 페이지에서 해당 방향 버튼 비활성.
- **데이터 바인딩 대상 API**:
  - `GET /memos?page=<n>&size=20` — 목록 페이징 (`createdAt DESC` 고정).

### 2. MemoDialogContainer (모달 3단계 흐름)

> `MemoListPage` 위에 겹쳐 표시되는 오버레이 컨테이너. 독립 라우트 없음. 한 개의 오버레이 위에서 "상세 → 편집 → 삭제 확인" 세 모드가 전환된다. 애니메이션은 과하지 않게, 같은 오버레이 컨테이너를 재활용하는 의도.

- **페이지 이름**: `MemoDialogContainer` (오버레이 컨테이너 — 세 모드를 호스팅)
- **Route**: 없음 (목록 페이지 위에 오버레이)
- **목적**: 메모 CRUD 의 상세 / 편집 / 삭제 확인 3단계 흐름을 하나의 오버레이에서 모드 전환으로 처리.
- **공통 UI 카테고리** (구현체 언급 금지):
  - **오버레이 컨테이너 (모달 유형)**: 화면 중앙 배치, 배경 딤 처리, ESC / 바깥 클릭으로 닫기.
  - 헤더 영역: 모드별 제목 + 우측 닫기 아이콘 버튼.
  - 푸터 영역: 모드별 액션 버튼 묶음 (우측 정렬).

#### 2.a. 상세 모드 (MemoDetailDialog)

- **진입 트리거**: 목록 카드 클릭.
- **필요 UI 카테고리**:
  - 모달 헤더 제목: 메모 제목 (긴 경우 말줄임 또는 줄바꿈 허용).
  - 본문 영역: **읽기 전용 plain text 블록**. 줄바꿈 유지, 마크다운/HTML 렌더 없음. 스크롤 가능한 긴 본문 대응.
  - 메타 라인: 작성일 / 수정일 (보조 톤 텍스트).
  - 푸터 액션:
    - Secondary 버튼 "삭제" (위험 톤 허용) → **2.c 삭제 확인 모드로 전환**.
    - Primary 버튼 "편집" → **2.b 편집 모드로 전환**.
- **주요 인터랙션**:
  - "편집" 클릭 → 같은 오버레이 안에서 편집 모드로 전환.
  - "삭제" 클릭 → 같은 오버레이 안에서 삭제 확인 모드로 전환.
  - 닫기 아이콘 / ESC / 배경 클릭 → 오버레이 닫고 목록으로 복귀.
- **데이터 바인딩 대상 API**:
  - `GET /memos/{id}` — 단건 조회 (카드에 이미 제목/본문 있으면 prefetch 된 값 사용도 가능, 최신 보증용으로 재조회 허용).

#### 2.b. 편집 / 신규 작성 모드 (MemoFormDialog)

- **진입 트리거**:
  - 헤더의 "+ 새 메모" 버튼 → 신규 작성 (빈 폼).
  - 상세 모드의 "편집" 버튼 → 기존 값 주입된 편집 폼.
- **필요 UI 카테고리**:
  - 모달 헤더 제목: 모드에 따라 "새 메모" 또는 "메모 편집".
  - 폼 본문:
    - **제목 필드** — 단일행 텍스트 입력. 라벨 "제목", 최대 100자. 필수 표시. 공백만 입력 시 거부 피드백.
    - **본문 필드** — 다중행 텍스트 입력. 라벨 "본문", 최대 5000자. 선택 (빈 값 허용).
    - 유효성 피드백은 필드 하단에 인라인 표시 (에러 톤).
    - (선택) 본문 글자수 카운터 — 보조 톤 텍스트.
  - 푸터 액션:
    - Secondary 버튼 "취소" → 변경사항 확인 없이 오버레이 닫기 (단순화 — V1 은 dirty-check 없음).
    - Primary 버튼 "저장" → 제출. 제출 중엔 버튼 로딩/비활성.
- **주요 인터랙션**:
  - 제목 비어있거나 공백만 → 저장 버튼 클릭 시 인라인 에러 노출, 서버 호출 미발생.
  - 저장 성공 → 오버레이 닫고 목록 새로고침. (편집 후면 상세 모드로 복귀하는 대신 목록으로 복귀 — 3단계 흐름은 "상세 → 편집 → 저장 후 목록" 방향이 기본 의도.)
  - 저장 실패 → 인라인 에러 또는 폼 상단 배너로 서버 에러 안내 (스택 트레이스 노출 금지, 한국어 문구만).
- **데이터 바인딩 대상 API**:
  - `POST /memos` — 신규 작성.
  - `PUT /memos/{id}` — 전체 교체 (title, content 둘 다 payload 포함 필수).

#### 2.c. 삭제 확인 모드 (MemoDeleteConfirmDialog)

- **진입 트리거**: 상세 모드의 "삭제" 버튼.
- **필요 UI 카테고리**:
  - 모달 헤더 제목: "메모 삭제".
  - 본문: 확인 안내 문구 + 대상 메모 제목 강조 표기. 복구 불가 고지 ("삭제한 메모는 복구할 수 없습니다" 류).
  - 푸터 액션:
    - Secondary 버튼 "취소" → 상세 모드로 복귀 (또는 오버레이 닫기).
    - Primary(위험 톤) 버튼 "삭제" → 서버 삭제 실행. 진행 중 로딩/비활성.
- **주요 인터랙션**:
  - "삭제" 클릭 → `DELETE` 호출. 성공 시 오버레이 닫고 목록 새로고침.
  - "취소" 클릭 → 상세 모드로 되돌리거나 오버레이 닫기 (UX 의도: 되돌리기 자연스러움 — "취소" 가 상세로 돌아가는 양보(escape) 경로).
- **데이터 바인딩 대상 API**:
  - `DELETE /memos/{id}`.

## 디자인 제약

- **토큰 기반** — 하드코딩 색/간격 금지. `libs/tokens` 스케일 및 Tailwind 토큰 유틸 경유.
- **Light / Dark 양쪽 대응** — 모든 상태 / 모드에서 두 테마 자동 대응. 단일 테마만 검증 금지.
- **헤더 유지 원칙** (`global-states.md` §2) — `MemoListPage` 는 Loading / Error / Empty 어느 상태에서도 페이지 헤더 ("메모" 제목 + "+ 새 메모" 버튼) 를 박탈하지 않는다. 본문 영역만 상태별로 치환.
- **목록 + 합성 카드 범주** (`data-display.md` §3) — 표(Table) 가 아니다. 행마다 제목/본문 스니펫/메타가 혼합되는 합성 카드 레이아웃. 고정 컬럼 격자 금지.
- **모달 3단계 흐름** — 상세 / 편집 / 삭제 확인은 독립 라우트가 아니라 하나의 오버레이 컨테이너 안에서의 모드 전환. 전환 애니메이션은 절제 (과한 슬라이드/스프링 금지), 동일 컨테이너 재활용 의도.
- **본문은 plain text** — 마크다운 / HTML 렌더 없음. 줄바꿈은 유지하되 다른 포매팅 기호는 원문 그대로 텍스트로 렌더.
- **한글 IME 대응 필드**:
  - 메모 제목 (단일행 텍스트 입력).
  - 메모 본문 (다중행 텍스트 입력).
  둘 다 조합 중 문자 (composition) 입력을 깨뜨리지 않는 방식이어야 함.
- **페이지네이션** — 기본 size=20. 목록 하단 "이전 / 다음" 방향 버튼 + `현재/총` 인디케이터. `createdAt DESC` 고정 정렬.
- **에러 문구** — 사용자에게 보이는 에러는 번역 가능한 한국어 문구만. 스택 트레이스 / 내부 에러 원문 노출 금지 (`global-states.md` §3.2).

## 비포함 (Out of scope)

Stitch 가 아래 기능을 추가로 생성하면 무시하고 구현 단계에서 제거할 것:

- 마크다운 프리뷰 / 렌더링.
- 첨부파일, 이미지 업로드, 이미지 썸네일.
- 검색, 텍스트 필터, 하이라이트.
- 태그 / 카테고리 / 라벨.
- 공유, 공개 링크, 초대.
- 정렬 옵션 UI (정렬은 `createdAt DESC` 고정, 사용자 선택 없음).
- 대량 선택 / 일괄 삭제.
- 메모 간 내비게이션 (이전 메모 / 다음 메모 버튼).
- 독립 라우트 형태의 편집 화면 (`/memos/new`, `/memos/:id/edit`) — 모달 오버레이가 유일 경로.
- 다국어 전환 UI (V1 은 한국어 고정).
- 사용자 / 소유자 개념 표시 (V1 은 단일 사용자 모델).
