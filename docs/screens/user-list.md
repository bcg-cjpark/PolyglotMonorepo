# UserListPage

## 기본 정보

- **Route**: `/users`
- **파일 위치**: `apps/example-web/src/pages/UserListPage.tsx`
- **관련 PRD**: [prd/user.md](../prd/user.md)
- **시안 선택**: UserListPage Variant A (표 중심) + UserFormPage Variant C (모달형) — [design-notes/user-variants.md](../design-notes/user-variants.md)

## 목적

등록된 사용자를 최신 생성순으로 조회하고, 신규 생성(모달 흐름) 진입점과 행 단위 삭제를 제공한다. 사용자 관리가 이 한 페이지에서 완결된다.

## 레이아웃

위에서 아래로:

- 페이지 헤더 — 왼쪽에 제목 "사용자", 오른쪽에 주요 액션(Primary) 버튼 "+ 새 사용자".
- 본문 — 표(정형 스키마). 컬럼 4개: 이름 / 이메일 / 생성일 / 행 단위 삭제 액션.
- (오버레이) 오버레이 컨테이너(모달 유형) — "+ 새 사용자" 클릭 시 본문 위에 뜸. 제목 "새 사용자", 본문 폼(세로 스택), 하단 액션 바(Primary "저장" + 보조 "취소").

헤더는 Loading / Error / Empty 상태 모두에서 항상 유지. 본문만 상태별 뷰로 치환.

## 컴포넌트

| 역할 | UI 종류 | 비고 |
|---|---|---|
| 페이지 헤더 | 제목 + 주요 액션 버튼 영역 | 제목은 페이지 탑 레벨 텍스트 |
| 주요 액션 (목록 레이어) | Primary 버튼 | 라벨 "+ 새 사용자". 클릭 시 오버레이 컨테이너 오픈 |
| 본문 표 | 표 (정형 스키마) | 4 컬럼, 중·소량 데이터. 자체 빈 상태 표시 기능 내장 |
| 행 내 삭제 | 파괴형(Destructive) 보조 버튼 | 표의 셀 안에 주입 |
| Loading 플레이스홀더 | 로딩 플레이스홀더 (리스트 형태) | 본문 자리 치환. 헤더는 유지 |
| Error 안내 | 한국어 에러 텍스트 + 보조 버튼 "다시 시도" | 본문 자리 치환. 스택 트레이스 노출 금지 |
| Empty 안내 | 표 내장 빈 상태 문구 | "등록된 사용자가 없습니다." |
| 오버레이 컨테이너 | 오버레이 컨테이너 (모달 유형) | 모달 헤더(제목 + 닫기) + 본문 + 하단 액션 바 |
| 모달 필드 × 2 | 단일행 텍스트 입력 | email (IME 비활성 전제), name (한글 IME 허용) |
| 모달 필드 보조 텍스트 | 라벨 + 헬퍼 / 에러 텍스트 | 각 입력 아래 |
| 모달 주요 액션 (오버레이 레이어) | Primary 버튼 | "저장" |
| 모달 보조 액션 | 보조 액션 버튼 (ghost/outline) | "취소" |

※ 구현 라이브러리 선택은 `libs/ui` 우선 규칙(`CLAUDE.md`) 에 따라 결정. 이 문서는 UI 카테고리만 명시.

## 인터랙션

1. 화면 진입 시 `GET /users` 호출 → 응답을 표 본문에 렌더 (createdAt DESC 정렬).
2. **Loading 중**: 페이지 헤더 유지. 본문은 로딩 플레이스홀더로 치환.
3. **Error**: 페이지 헤더 유지. 본문에 한국어 에러 텍스트 + "다시 시도" 버튼. "다시 시도" 클릭 시 `GET /users` 재호출.
4. **Empty (목록 비어있음)**: 페이지 헤더 유지. 표의 빈 상태 자리에 "등록된 사용자가 없습니다." 문구. 사용자는 헤더의 "+ 새 사용자" 로 생성.
5. "+ 새 사용자" 버튼 클릭 → 오버레이 컨테이너(모달) 오픈. 포커스는 첫 입력(email)로 이동.
6. 모달 내부에서 email / name 입력 → "저장" 버튼 클릭:
   - 성공 (`POST /users` → 201): 모달 닫힘 + 목록 재조회 (`GET /users` invalidate). 신규 사용자가 목록 상단에 보임.
   - 이메일 중복 (`POST /users` → 409): 모달 유지. email 필드 하단 에러 텍스트 "이미 사용 중인 이메일입니다."
   - 유효성 실패 (로컬 검증): 해당 필드 하단 에러 텍스트 — 이메일 형식 위반 → "이메일 형식이 올바르지 않습니다.", name 공백만 → "이름을 입력하세요."
   - 저장 중 상태: "저장" 버튼 로딩/비활성화. "취소" 는 계속 가능.
7. 모달 닫기 경로 (어느 것이든 닫힘, 입력값 폐기):
   - "취소" 버튼 클릭
   - ESC 키
   - 오버레이(배경) 클릭
8. 행 우측 "삭제" 버튼 클릭 → 확인 프롬프트 (브라우저 네이티브 `confirm` 또는 확인용 오버레이 컨테이너) → 확인 시 `DELETE /users/{id}` 호출 → 성공 (204) 시 목록 재조회. 실패 (404) 시 한국어 에러 안내.

## 데이터 바인딩

- `GET /users` — 목록 조회. 응답을 본문 표에 바인딩. 로딩/에러/빈 상태는 위 인터랙션 3~4 에 따라 표시.
- `POST /users` — 모달 내부 "저장". Request body `{ email, name }`. 응답 201 / 409 / 400 중 분기. 성공 후 목록 쿼리 무효화로 자동 재조회.
- `DELETE /users/{id}` — 행 삭제. 성공 (204) 후 목록 쿼리 무효화.

에러 문구는 한국어 고정. 서버 원문 / 스택 트레이스 노출 금지.

## 상태 (State)

| 상태명 | 타입 | 초기값 | 업데이트 트리거 |
|---|---|---|---|
| users | User[] | [] | `GET /users` 응답 |
| isLoading | boolean | true | 쿼리 로딩 상태 |
| isError | boolean | false | 쿼리 실패 |
| isModalOpen | boolean | false | "+ 새 사용자" 클릭 / 저장 성공 / 취소·ESC·오버레이 클릭 |
| formEmail | string | "" | 모달 입력 |
| formName | string | "" | 모달 입력 |
| formErrors | { email?: string; name?: string } | {} | 로컬 검증 실패 / 409 응답 |
| isSubmitting | boolean | false | 저장 요청 중 |

모달이 닫힐 때 `formEmail`, `formName`, `formErrors`, `isSubmitting` 은 초기화.
