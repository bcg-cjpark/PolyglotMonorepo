- **파일 위치**: `apps/example-web/src/pages/MemoListPage.tsx` 내 렌더 + `apps/example-web/src/components/Memo*Dialog.tsx` 합성 (페이지 전용, `libs/ui` 아님)

## 목적

메모의 상세 열람 / 편집·신규 작성 / 삭제 확인 3 모드를 **하나의 오버레이 컨테이너** 안에서 모드 전환으로 처리. 전환 애니메이션은 절제, 동일 컨테이너 재활용 의도.

## 레이아웃

모드 공통 — 오버레이 컨테이너(화면 중앙 모달, 배경 딤):

- 모달 헤더 — 모드별 제목 + 우측 닫기 아이콘 버튼.
- 모달 본문 — 모드별 콘텐츠.
- 모달 푸터 — 모드별 액션 버튼 묶음 (우측 정렬).

## 모드별 세부

### 2.a. 상세 모드

- **진입 트리거**: [memo-list.md](./memo-list.md) 의 카드 클릭.
- **헤더 제목**: 메모 제목 (긴 경우 줄바꿈 또는 말줄임).
- **본문**: 읽기 전용 plain text 블록. 줄바꿈 유지 (`white-space: pre-wrap` 의미). 마크다운/HTML 렌더 금지. 본문이 길면 스크롤.
- **메타**: 작성일 / 수정일 (보조 톤).
- **푸터 액션**:
  - 파괴형 보조 "삭제" — 삭제 확인 모드로 전환.
  - Primary "편집" — 편집 모드로 전환.

### 2.b. 편집 / 신규 작성 모드

- **진입 트리거**:
  - 헤더 "+ 새 메모" → 신규 (빈 폼)
  - 상세 모드의 "편집" → 기존 값 주입
- **헤더 제목**: "새 메모" / "메모 편집".
- **본문 (폼, 세로 스택)**:
  - 제목 입력 — 단일행 텍스트. 라벨 "제목". 필수. 최대 100자. 공백만 금지. 한글 IME.
  - 본문 입력 — 다중행 텍스트. 라벨 "본문". 선택 (빈 값 허용). 최대 5000자. 한글 IME.
  - 각 필드 아래 에러/헬퍼 텍스트 자리.
  - (선택) 본문 글자수 카운터 (보조 톤).
- **푸터 액션**:
  - 보조 "취소" — 변경사항 확인 없이 닫기 (V1 dirty-check 없음).
  - Primary "저장" — 제출. 중간엔 로딩/비활성.

### 2.c. 삭제 확인 모드

- **진입 트리거**: 상세 모드의 "삭제" 버튼.
- **헤더 제목**: "메모 삭제".
- **본문**: 확인 안내 문구 + 대상 메모 제목 강조 표기. 복구 불가 고지 ("삭제한 메모는 복구할 수 없습니다").
- **푸터 액션**:
  - 보조 "취소" — 상세 모드로 양보(escape) 복귀.
  - 파괴형 Primary "삭제" — 서버 삭제 실행. 중간엔 로딩/비활성.

## 인터랙션

### 공통
- 닫기 아이콘 / ESC / 배경 클릭 → 오버레이 닫기, 목록으로 복귀 (dialogState `"none"`).

### 상세 모드
1. 진입 시 `GET /memos/{id}` 호출 (카드에 이미 있는 값을 prefetch 로 쓸 수도 있음, 최신 보증 용도).
2. "편집" 클릭 → 같은 오버레이 안에서 편집 모드로 전환.
3. "삭제" 클릭 → 같은 오버레이 안에서 삭제 확인 모드로 전환.

### 편집/신규 모드
1. 제목 공백만 / 100자 초과 → 인라인 에러, 저장 버튼 비활성.
2. 본문 5000자 초과 → 인라인 에러.
3. "저장" 클릭:
   - 신규: `POST /memos {title, content}` → 201 → 오버레이 닫기 + 목록 새로고침.
   - 편집: `PUT /memos/{id} {title, content}` (전체 교체, content 는 null 또는 "" 허용) → 200 → 오버레이 닫기 + 목록 새로고침.
   - 400/기타 에러: 인라인 또는 폼 상단 배너로 한국어 안내.
4. "취소" 클릭 → 확인 없이 닫기.

### 삭제 확인 모드
1. "삭제" 클릭 → `DELETE /memos/{id}` → 204 → 오버레이 닫기 + 목록 새로고침.
2. "취소" 클릭 → 상세 모드로 되돌리기 (양보 경로).

## 데이터 바인딩

- `GET /memos/{id}` — 상세 단건 (재조회).
- `POST /memos` — 신규 `{title, content}`.
- `PUT /memos/{id}` — 전체 교체 `{title, content}`.
- `DELETE /memos/{id}` — 삭제.

성공 후 목록 쿼리 (`['memos', {page, size}]`) 무효화로 자동 재조회.
